# 비주얼 리뷰 보고서

## 종합 평가
- **개발 준비 상태**: 🟢 준비 완료
- **총평**: 코지 픽셀아트 스타일과 따뜻한 컬러 팔레트가 게임의 "이미 괜찮은 사람의 다음 무대" 톤에 정확히 부합. 모바일 퍼스트 UI가 출퇴근 사용 시나리오에 적합.

## 발견 사항

### 🟡 권장 수정

1. **09_layout.html은 Team 3에서 실제 구현 시 제작**
   - 현재: 아트 디렉션과 UI 텍스트로 설계 완료
   - 제안: Team 3 frontend-dev가 Next.js로 직접 구현. 별도 정적 HTML 불필요
   - 이유: 동적 UI(스탯 변화, 별이 대사, 시간대별 변화)는 정적 HTML로 불가

2. **다크모드 컬러 팔레트 상세화**
   - 현재: "차콜 배경으로 전환"만 명시
   - 제안: 다크모드 전용 팔레트 추가 (차콜#2C3E50, 다크크림#3D3A30, 소프트골드#D4B94E)

3. **픽셀아트 에셋 제작 우선순위 명확화**
   - 제안 순서: ①별이 → ②메인캐릭터 → ③프리 → ④구역배경 → ⑤감독들
   - 이유: 별이와 캐릭터가 대시보드 메인이라 가장 먼저 필요

### 🟢 참고 사항

1. **Galmuri11 폰트** 선택이 적절. 픽셀아트와 한글의 조화가 좋고 무료 사용 가능

2. **시간대별 비주얼 변화**가 앱의 "살아있는" 느낌을 줄 핵심 요소. 구현 우선순위 높음

3. **"원탭 체크" + 파티클 피드백** 조합이 Stardew Valley의 수확 쾌감과 유사. 게임성의 핵심

## 정합성 매트릭스

| 검증 항목 | 상태 | 비고 |
|----------|------|------|
| 아트 디렉션 ↔ 세계관 | ✅ | 5개 구역 컬러/분위기가 세계관과 정확히 매핑 |
| UI 텍스트 ↔ 대사 스크립트 | ✅ | 별이의 보이스가 UI 마이크로카피와 일관성 유지 |
| 이미지 프롬프트 ↔ 캐릭터 설정 | ✅ | 캐릭터 외형이 세계관 인물 설정과 일치 |
| 컬러 팔레트 ↔ 5대 스탯 | ✅ | 각 스탯에 고유 컬러 부여, 직관적 |
| 모바일 UI ↔ 유저 시나리오 | ✅ | 출퇴근 열차에서 한 손 사용 고려 |
| 전체 톤 ↔ 유저 피드백 | ✅ | "이미 괜찮은 사람"의 긍정적 여정 톤 반영 |

### 핵심 화면 목록
1. **대시보드** — 스탯, 퀘스트, 별이 대사, 구역 네비게이션
2. **구역 맵** — 5+2 구역 선택 화면
3. **퀘스트 상세** — 퀘스트 목표, 단계, 보상
4. **별이 대화** — 채팅 UI (OpenAI 연동)
5. **스탯 상세** — 각 스탯 히스토리, 그래프
6. **프로필** — 칭호, 업적, 플레이 기록
7. **온보딩** — 3단계 풀스크린 슬라이드

### 구현 우선순위
1. 대시보드 (데일리 퀘스트 체크 + 스탯 표시)
2. 별이 대화 (OpenAI 연동)
3. 스탯 추적 시스템
4. 구역 맵 + 퀘스트 상세
5. 온보딩
6. 프로필/칭호 시스템
